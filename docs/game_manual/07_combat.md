# 07 战斗系统

## 战斗定位

首版采用回合制战斗。战斗主轴是 **普通攻击、防御、躲避和 4 个职业技能**，状态卡只作为战斗中的临时修饰能力。玩家即使没有抽到理想状态卡，也必须能依靠基础行动和技能完成完整战斗。

状态卡可以提供行动倍率、确定暴击、强化躲避、强化防御或紧急回撤修饰，但不应直接替代核心行动。状态卡不是局外卡组，不再形成手牌；每回合开始自动抽取 1 个状态 Buff 并直接生效。战斗结算不使用直接躲避概率或直接暴击概率；概率只用于状态 Buff 抽取，例如抽到暴击状态 Buff 的基础概率为 5%。

战斗视角采用第一人称打牌视角。敌人以卡牌和文字信息展示在玩家前方，玩家的基础行动、技能、状态卡和装备主动效果位于界面下方。没有美术资源时，所有敌人都可以使用纯色卡牌、名称、特性、技能文本和意图标签表达。

玩家默认先手。若敌人拥有“先手”特性，则战斗开始时先结算该敌人的第 1 次基础意图，再进入玩家第 1 回合。

## 基础参数

| 参数 | 默认值 |
| --- | ---: |
| 每回合行动力 | 第 1 回合 1 点，第 2 回合 2 点，第 3 回合起 3 点 |
| 每回合状态卡抽取 | 1 张 |
| 状态 Buff 持有上限 | 1 个当前回合 Buff |
| 每回合状态卡使用上限 | 不再手动使用 |
| 每场战斗初始格挡值 | 0 |
| 状态卡抽取配置 | 每场战斗开始生成，战斗结束清空 |

## 核心行动

玩家每回合都可以使用以下固定行动。固定行动不依赖抽卡。

| 行动 | 消耗 | 效果 |
| --- | ---: | --- |
| 普通攻击 | 1 行动力 | 使用当前武器造成基础伤害，并触发命中、职业资源和装备效果。 |
| 防御 | 1 行动力 | 获得格挡值，用于吸收护甲减伤后的剩余伤害。 |
| 躲避 | 1 行动力 | 获得 1 层躲避，抵消下一次直接攻击的 1 段伤害。 |
| 职业技能 | 0-3 行动力 | 使用当前 4 个技能槽中的技能，受冷却、资源和融合影响。 |
| 装备主动 | 0-2 行动力 | 触发装备主动效果，通常有每战限制或冷却。 |

## 普通攻击

普通攻击是稳定输出来源。

基础规则：

- 伤害 = 攻击力 × 状态卡倍率 × 套装倍率 × 充能倍率，各乘法节点独立计算。
- 普通攻击不会自然暴击；只有使用暴击状态卡或套装提供的确定暴击效果后，下一次普通攻击或攻击技能才会暴击。
- 普通攻击可以触发装备命中特效和套装效果。
- 普通攻击不能被状态卡替代；状态卡只能提高下一次或本回合攻击表现。

示例状态卡：

- 平稳：下一次普通攻击或攻击技能按 100% 伤害结算。
- 效果不错：下一次普通攻击或攻击技能按 110% 伤害结算。
- 暴击：下一次普通攻击或攻击技能造成 200% 伤害。

## 防御

防御是稳定承伤手段。

基础规则：

- 防御行动获得格挡值，不直接增加常驻护甲。
- 护甲是角色常驻减伤属性，不会因为受击而被消耗。
- 受到攻击时先按护甲降低伤害，再用格挡值吸收剩余伤害，最后才扣生命。
- 护甲减伤公式：`剩余伤害 = 原始伤害 * 30 / (30 + 护甲值)`，最低造成 1 点伤害。
- 格挡值在同一场战斗内保留，受到攻击时被消耗，战斗结束后清零。

示例状态卡：

- 平稳：下一次防御按 100% 格挡效果结算。
- 效果拔群：下一次防御按 120% 格挡效果结算。
- 完美格挡：下一次防御按 200% 格挡效果结算。

## 躲避

躲避是确定型规避手段，适合弓箭手和轻甲构筑。

基础规则：

- 躲避行动获得 1 层躲避。
- 每 1 层躲避抵消下一次直接攻击的 1 段伤害。
- 多段攻击会逐段消耗躲避层数。
- 躲避不能避免中毒、燃烧等持续伤害。
- 回合结束时未使用的躲避层默认清空，除非有装备、技能或套装使其保留。

示例状态卡：

- 平稳：下一次躲避按 100% 躲避效果结算。
- 识破：下一次躲避按 200% 躲避效果结算；若基础为 1 层躲避，则获得 2 层。
- 紧急回撤：下一次攻击按 100% 攻击效果结算，并同时获得 50% 防御效果。

## 职业技能

每名角色同时装备 4 个技能。技能是职业战斗身份的核心，不进入状态卡抽取配置。

技能规则：

- 技能消耗行动力、职业资源或冷却。
- 技能可以被装备和融合增强；普通攻击、防御、躲避也可以承载本局内融合附着。
- 技能不会因为抽卡不顺而不可用，但可能受冷却或资源限制。
- 技能与普通攻击、防御、躲避共同构成完整战斗手段。

## 状态卡

状态卡是战斗中的临时修饰能力，不是玩家局外构筑的卡组。首版不再把状态卡抽到手牌中，而是每回合自动抽取 1 个状态 Buff。

状态卡规则：

- 每回合开始自动抽取 1 个状态 Buff。
- 当前只存在 1 个状态 Buff，新回合会替换上一回合未消耗的状态 Buff。
- 状态 Buff 不需要手动点击使用。
- 状态卡通常不直接造成大量伤害，不直接提供大量护甲。
- 平稳、效果不错、效果拔群这类通用倍率 Buff 在当前回合内持续修饰有数值的行动。
- 暴击、识破、完美格挡、紧急回撤这类强关联 Buff 只在对应行动上触发 1 次，触发后立即消耗。
- 状态 Buff 只在当前战斗内存在；战斗结束后清空。

通用状态卡抽取概率：

| 状态卡 | 基础抽取概率 | 效果 |
| --- | ---: | --- |
| 平稳 | 50% | 下一次有数值的基础行动或技能按 100% 效果结算。 |
| 效果不错 | 20% | 下一次有数值的基础行动或技能按 110% 效果结算。 |
| 效果拔群 | 10% | 下一次有数值的基础行动或技能按 120% 效果结算。 |
| 暴击 | 5% | 下一次普通攻击或攻击技能造成 200% 伤害。 |
| 识破 | 5% | 下一次躲避行动获得 200% 躲避效果。 |
| 完美格挡 | 5% | 下一次防御行动获得 200% 格挡效果。 |
| 紧急回撤 | 5% | 下一次攻击按 100% 攻击效果结算，并同时获得 50% 防御效果。 |

抽取概率修正规则：

- 基础抽取概率是全职业通用默认值，不和职业绑定。
- 装备、套装、技能、融合可以修改某一张状态卡的抽取概率。
- 暴击相关概率修正必须写成“暴击状态卡抽取概率 +X”，不代表攻击会自然暴击。
- 概率修正只影响状态卡抽取，不改变普通攻击、防御、躲避和技能的确定性结算。
- 概率修正后需要归一化，使最终抽取概率总和保持 100%。

状态 Buff 战斗记录：

- 状态卡抽取配置：本场战斗可抽取的状态卡类型和概率。
- 当前状态 Buff：本回合正在生效的状态修饰。
- 触发记录：本回合已触发并消耗的一次性状态 Buff。

## 回合流程

玩家回合开始：

1. 触发回合开始效果。
2. 获得行动力：第 1 回合 1 点，第 2 回合 2 点，第 3 回合起 3 点。
3. 抽取 1 个状态 Buff 并直接生效。
4. 显示怪物意图。

玩家行动：

1. 状态 Buff 自动修饰本回合行动或下一次对应行动。
2. 执行普通攻击、防御、躲避。
3. 使用 4 个技能槽中的职业技能。
4. 使用装备主动效果。
5. 结算融合附着触发。
6. 结束回合。

怪物回合：

1. 怪物按意图行动。
2. 结算躲避、护甲减伤、格挡值、生命伤害和反击。
3. 结算中毒、燃烧、流血等持续状态。
4. 清除本回合状态卡修饰；格挡值保留到战斗结束或被攻击消耗。
5. 进入下一玩家回合。

## 怪物意图

怪物行动前必须向玩家展示意图。常见意图：

- 攻击：显示预计伤害和攻击次数。
- 防御：获得护甲或减伤。
- 强化：提高自身属性。
- 弱化：给玩家施加负面状态。
- 召唤：生成小怪。
- 特殊：触发怪物机制。

## 常用状态

### 伤害类型

当前原型支持以下伤害类型：

| 类型 | 常量 | 说明 |
| --- | --- | --- |
| 物理 | PHYSICAL | 默认伤害类型。 |
| 火焰 | FIRE | 火焰伤害。 |
| 冰冻 | ICE | 冰冻伤害。 |
| 闪电 | LIGHTNING | 闪电伤害。 |
| 毒素 | POISON | 毒素伤害。 |
| 神圣 | HOLY | 神圣伤害。 |
| 暗影 | SHADOW | 暗影伤害。 |
| 真实 | TRUE | 真实伤害，无视抗性。 |

目标可通过 `resistances` 字典配置抗性系数，例如 `{"fire": 0.5, "ice": 1.5}`。抗性系数与 status 的 `resist_*` 效果合并计算。

### 正面状态

| 状态 | 效果 |
| --- | --- |
| 力量 | 攻击力提高（通过 status 的 percent/multiply 效果）。 |
| 坚韧 | 防御/格挡值提高。 |
| 减伤 | 受到伤害按固定百分比降低。 |
| 护甲 | 常驻减伤属性，降低每次直接攻击的伤害，不被消耗。 |
| 格挡值 | 战斗内临时防护值，吸收护甲减伤后的剩余伤害，战斗结束后清零。 |
| 躲避层 | 每层抵消下一次直接攻击的 1 段伤害。 |

负面状态：

| 状态 | 效果 |
| --- | --- |
| 破甲 | 受到攻击伤害提高。 |
| 标记 | 受到弓箭手相关伤害提高，造成伤害降低。 |
| 虚弱 | 造成伤害降低。 |
| 易伤 | 受到伤害提高。 |
| 中毒 | 回合开始受到持续伤害（通过 DOT 触发器实现）。 |

### Status 效果类型

Status 通过以下效果类型修改属性：

| 类型 | 常量 | 公式 |
| --- | --- | --- |
| 固定值 | flat | `base + flat_sum` |
| 百分比 | percent | `* (1 + percent_product)` |
| 乘法 | multiply | `* multiply_product` |

最终公式：`(base + flat_sum) * percent_product * multiply_product`

### Status 触发器

Status 可通过 `triggers` 配置在特定事件触发动作：

| 事件 | 触发时机 |
| --- | --- |
| on_turn_start | 回合开始 |
| on_turn_end | 回合结束 |
| on_hit_dealt | 造成命中后 |
| on_hit_received | 受到命中后 |
| on_dodge | 闪避成功时 |
| on_kill | 击杀目标时 |
| on_critical | 暴击时 |
| on_block_gain | 获得格挡时 |
| on_heal_received | 受到治疗时 |
| on_battle_start | 战斗开始时 |

动作类型包括：dot（持续伤害）、hot（持续回复）、reflect（反弹）、lifesteal（吸血）、gain_block（获得格挡）、gain_dodge（获得躲避）、heal（治疗）、apply_status（施加状态）、remove_status（移除状态）、extra_damage（额外伤害）、counter_all（AOE反击）、increment_counter（计数器递增）、reset_counter（计数器重置）。

## 胜负条件

胜利：

- 当前战斗中所有敌人生命降至 0。
- 进入战斗胜利奖励界面。

失败：

- 玩家生命降至 0。
- 当前爬塔局结束。
- 清空塔内附着卡和融合附着。
