# 15 开发任务拆解

## 首版开发目标

首版原型目标是验证核心战斗和爬塔循环，不依赖美术资源。所有角色、敌人、装备、技能、状态卡和奖励都可以使用卡牌、图标占位、文本说明和数值条表达。

视角采用第一人称打牌视角：

- 屏幕上方：敌人卡牌、生命、护甲、意图、特性。
- 屏幕中部：战斗日志、玩家状态、当前层数和第几场战斗。
- 屏幕下方：普通攻击、防御、躲避、4 个技能、当前持有状态卡。
- 右侧或弹层：装备、塔内附着、套装进度和敌人详情。

玩家默认先手。若敌人拥有“先手”特性，则进入战斗后先结算敌人的第 1 次基础意图，再进入玩家回合。

## 阶段 1：Godot 工程骨架

目标：创建可运行的 Godot 4.x 项目，进入主战斗场景。

任务：

1. 创建 `GameProject/`。
2. 创建主菜单、战斗场景、奖励场景和数据加载入口。
3. 建立基础 UI 主题：卡牌、按钮、状态条、意图标签。
4. 使用纯色卡牌和文字占位，不接入美术。

验收：

- 启动后能进入一场测试战斗。
- PC 分辨率和移动端竖屏布局都不遮挡核心按钮。

## 阶段 2：数据驱动

目标：把职业、状态卡、敌人、装备和技能都做成数据表。

任务：

1. 定义 JSON 数据结构：`classes`、`state_cards`、`enemies`、`enemy_traits`、`enemy_skills`、`equipment`、`skills`。
2. 实现数据加载、ID 校验和缺字段报错。
3. 先录入通用状态卡 7 张。
4. 录入 50 个敌人的基础数据和系数。

验收：

- 游戏启动时能加载所有数据。
- 敌人可按层数计算生命、攻击、防御。
- 缺少敌人技能或特性 ID 时能在调试日志报错。

## 阶段 3：单场战斗核心

目标：完成一个可玩的回合制战斗。

任务：

1. 实现玩家属性：生命、护甲、攻击、行动力、状态卡持有。
2. 实现基础行动：普通攻击、防御、躲避。
3. 实现状态卡抽取：平稳、效果不错、效果拔群、暴击、识破、完美格挡、紧急回撤。
4. 实现敌人意图：攻击、防御、技能、召唤、强化。
5. 实现先手特性。
6. 实现胜负条件。

验收：

- 玩家每回合抽 2 张状态卡。
- 状态卡概率符合通用状态卡池。
- 敌人有先手时会先行动。
- 玩家能通过基础行动击败测试敌人。

## 阶段 4：敌人系统

目标：把 50 种敌人接入战斗。

任务：

1. 实现敌人特性系统。
2. 实现敌人技能系统。
3. 实现普通、精英、Boss 的数值倍率。
4. 实现每层 7 普通、2 精英、1 Boss 的战斗队列。
5. 实现 Boss 技能轮转和生命阈值技能。

验收：

- 50 种敌人都能生成。
- 普通敌人只有 1 个特性。
- 精英敌人有 2 个特性和 1-2 个技能。
- Boss 有 3 个特性和 3 个技能。
- 层数提高后敌人的生命、攻击、防御明显提高。

## 阶段 5：奖励与局内成长

目标：完成最小爬塔循环。

任务：

1. 实现普通怪 3 选 1、精英怪 4 选 1、Boss 非装备卡 5 选 1。
2. 实现 Boss 三分支奖励：非装备卡、永久装备、技能。
3. 实现塔内附着卡立即附着。
4. 实现状态卡概率强化。
5. 实现第 1 层新手引导固定解锁。

验收：

- 普通怪和精英怪不出现装备卡。
- Boss 永久装备分支可以解锁装备。
- 塔内附着卡不能暂存，必须立即选择目标。
- 新游戏会清空塔内附着。

## 阶段 6：装备、技能和套装

目标：让构筑开始成立。

任务：

1. 实现 12 个装备位置。
2. 实现每职业 4 个技能槽。
3. 实现普通装备和套装装备。
4. 实现 2/3/4/5/6 件套效果。
5. 实现清辉/流霜对戒的套装门槛 -1。

验收：

- 装备提供生命、攻击、护甲三属性。
- 戒指可装备 2 个。
- 套装进度正确显示并触发。
- 技能不会重复获取。

## 阶段 7：可玩性验证

目标：验证“只有成功构筑才能上高层”。

任务：

1. 制作 10 层测试曲线。
2. 调整敌人公式、奖励强度和状态卡概率。
3. 记录每层失败原因。
4. 调整普通、精英、Boss 的压迫节奏。

验收：

- 无构筑随机点击难以稳定通过第 10 层。
- 正确使用装备、技能、状态卡和塔内附着可以明显推进更高层。
- 玩家死亡原因能从战斗日志中看懂。
