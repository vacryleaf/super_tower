# 12 首版装备生成

## 生成目标

首版装备池用于支撑第 1 层新手引导固定解锁和 Boss 永久装备奖励。正式装备卡只在 Boss 关卡出现，不会出现在普通怪、强化怪、精英怪、商店或事件奖励中。装备不按强弱稀有度划分，只分为普通装备和套装装备。

生成原则：

- 普通装备提供更高基础数值，单件即可稳定提升角色。
- 套装装备基础数值较低，但通过套装件数提供特殊能力。
- 所有装备卡都必须提供生命、攻击、护甲三项基础属性；三项都需要显式填写，允许为 0，不允许缺项。
- 普通装备的三属性总量高于同栏位套装装备；套装装备的价值主要由套装效果承担。
- 套装效果以规则变化为主，必须改变至少一个系统层面，包括基础行动、状态卡抽取、技能冷却、附着联动、奖励选择、怪物意图交互或职业资源流向。
- 直接伤害、护甲、生命等数值提升只能作为套装副效果，不能成为套装的唯一价值。
- 新手引导只固定解锁头部、上身、腰部、下身、手部、护腿、脚部、武器、副手，不解锁项链和戒指。
- 项链和戒指没有基础普通装备，只能作为套装部件出现。
- Boss 永久装备可以提供普通装备或套装装备。
- 套装装备分为职业套装和通用套装；通用套装不限职业，战士和弓箭手都可以装备并触发。
- 普通怪、强化怪、精英怪、商店和事件不生成装备卡。
- 套装生成应优先服务补全：当玩家已有某套装 1/2/3/4/5 件时，优先提供能触发下一档套装能力的缺失栏位。
- 每个套装部件都必须拥有独立装备名，不能只记录套装名和栏位。
- 本文件的套装表用于展示套装效果汇总；每件套装装备的独立基础属性见 [13_set_equipment_items.md](13_set_equipment_items.md)。

低效打击、低效复制和低效版本统一指：复制原行动的标签、目标和触发关系，但基础数值按原效果的 50% 结算，且不再次触发状态卡抽取。

## 生成流程

装备奖励按以下步骤生成：

1. 确定装备来源：新手引导固定解锁或 Boss 永久装备。
2. 确定装备类型：普通装备或套装装备。
3. 确定目标栏位：优先补足空栏位，其次参考职业倾向和当前构筑标签。
4. 若目标栏位是项链或戒指，强制生成套装装备。
5. 若生成套装装备，先选择套装，再选择该套装中可用且未装备的栏位。
6. 若玩家已有 1/2/3/4/5 件同套装，优先生成能触发下一档套装能力的缺失部件。
7. 生成装备实例，装备实例可在塔内承载融合附着。

Boss 永久装备候选优先级：

| 候选情况 | 处理规则 |
| --- | --- |
| 新手引导 | 固定解锁 9 件普通基础装备，不进入正式候选池。 |
| 项链或戒指栏位 | 只生成套装装备。 |
| 已有套装接近下一档能力 | 优先生成该套装缺失部件。 |
| 非项链、非戒指栏位且没有合适套装补全部件 | 可以生成普通装备、职业套装起点或通用套装起点。 |

若 Boss 永久装备分支的候选栏位是项链或戒指，则固定生成套装装备。

套装补全优先级：

| 当前同套装件数 | 补全目标 |
| ---: | --- |
| 1 件 | 优先提供该 2 件套或更大套装的第 2 件。 |
| 2 件 | 优先提供 3 件套或更大套装的第 3 件。 |
| 3 件 | 优先提供 4 件套或更大套装的第 4 件。 |
| 4 件 | 优先提供 5 件套或 6 件套的第 5 件。 |
| 5 件 | 优先提供 6 件套的第 6 件。 |
